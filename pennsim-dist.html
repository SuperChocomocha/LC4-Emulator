This document summarizes the known bugs in PennSim. If you encounter a bug or oddity that is not mentioned below, create a thread on the course discussion board with a detailed error report.

Current Bugs/Notes

  • Keyboard input is not received unless you currently have the ASCII display selected in PennSim. If your program seems to not be responding to input, make sure that the ASCII display box is highlighted yellow, and then try again. This is true for any field in PennSim: make sure that the intended area is selected before entering any values.

  • PennSim currently does not run at full speed unless the memory viewer is scrolled all of the way to the top. To fix this, select the scroll bar and drag it all the way up until you can see the value at address x0000.

  • In order for the display to reset each pixel to black automatically, PennSim must be run in double-buffered mode. This is accomplished by running PennSim with a '-d' parameter added. On Windows, create a shortcut to PennSim.jar, and then modify the Target line to include the '-d'.

  • You cannot currently set a breakpoint using the command 'break set (address)'; you can only specify a label to the command line. If the address does not have an associated label, you must set the breakpoint manually (scroll to the address, and check the “BP” box).

  • If you set a breakpoint on a label, and then reset PennSim (via the 'reset' command), the breakpoint will still appear as a little red box on the memory viewer scroll bar, but it will not be active. There is no known way to remove this red box (even by toggling the breakpoint) without restarting PennSim, but it can be ignored.

Changelog

  • Added "check cumulative" command to query the number of checks since the last "check reset" command was issue. The simulator resets the non-cumulative counts, but not the cumulative counts.

  • Added more useful feedback for some uncommon hardware events (memory protection fault, for example)

  • Prevented user from trying to load object files that didn't end in .obj (a common mistake)
